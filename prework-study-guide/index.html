HTML

  • An HTML is a hypertext markup language use to create web applications
  • The HTML file has two parts: the head and the body
  • The head usually contains information not visible to the end user
  • The body contains the visble part of the website - the content
  • Elements are used to add meaning to the HTML language
  • Tags tend to allow web developers to add specific function to the code
  • HTML provides the framework for the website or web application

CSS

  • CSS stands for Cascading Style Sheets
  • CSS provides style and aesthetics to the page
  • The CSS file can be in a different folder and linked to the HTML file
  • This is called external CSS

GIT

  • GIT is the local "memory" for the project we are working on
  • We create a local repository in a folder that is synced with the Github Repo online
  • After, the Xcode interface will be linked to the Github account through an SSH key
  • The files you work on will be added to the stagging phase, then commited, then finllay pushed up to Github
  • GIT is a version control system (VCS)
  • The Commit part time-stamps the progress you have made on your files
  • Branching and merging are the main tenets of VCS
  • Using git commnads such as git status tells you what branch you are currently working on
  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • JavaScript gives the web page functionality and interactivity
  • JavaScript provides the enduser the ability to input content
  • Javascript files end in .js
